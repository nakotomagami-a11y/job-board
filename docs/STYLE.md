# Style rules for application text

Text that will be pasted into a job application form (cover-letter answers, "why us?" essays, free-text custom questions) must read like it was typed by a human, not generated by an LLM.

## Punctuation

- Use `-` (hyphen). Never `—` (em-dash) or `–` (en-dash). Em-dashes are a known LLM tell.
- Use plain quotes `"..."`, not curly quotes `"..."`.
- Use `...` (three dots), not `…` (ellipsis character).

## Word choice

Avoid: *delve, tapestry, leverage (as verb), navigate (metaphorically), embark on, in the realm of, in today's fast-paced, moreover, furthermore, plethora, myriad, robust (as filler), seamless (as filler), unlock (as verb)*.

Prefer the simple verb. "Use" beats "leverage." "Many" beats "a plethora of." "Strong" beats "robust" when you mean strong.

## Where this applies

- Any value stored in `data/user/answer-bank.json`
- Any text suggested by the matcher agent that ends up in a form field
- Any drafted answer the business-strategist or developer agent produces

## Where this does NOT apply

- Code comments
- Documentation
- Strategy memos in chat
- Bug reports

Reason: those texts never reach a recruiter. The rule is about not flagging the user's applications as AI-generated.
